# Design: Автоматическое закрытие модального окна при входе в комнату

**Дата:** 2025-01-08  
**Автор:** OpenCode  
**Статус:** Утверждено

## Описание

При подключении пользователя к комнате (создании или присоединении) модальное окно для ввода кода комнаты автоматически закрывается.

## Проблема

Текущее поведение: когда пользователь вводит код комнаты и успешно подключается, модальное окно остается открытым поверх экрана комнаты, что создает неудобства и требует дополнительного взаимодействия для его закрытия.

## Решение

Добавить вызов функции `closeModal()` в начало функции `enterRoom(code)` в файле `public/app.js`.

### Изменения в коде

**Файл:** `public/app.js`  
**Функция:** `enterRoom(code)` (строки 465-485)

**До:**
```javascript
function enterRoom(code) {
  state.roomCode = code;
  $('room-code').textContent = code;
  screens.room();
  
  getLocalMedia().then((hasMedia) => {
    // ...
  });
}
```

**После:**
```javascript
function enterRoom(code) {
  closeModal(); // Закрываем модальное окно если оно открыто
  state.roomCode = code;
  $('room-code').textContent = code;
  screens.room();
  
  getLocalMedia().then((hasMedia) => {
    // ...
  });
}
```

## Поведение

### Успешные сценарии

1. **Создание комнаты:**
   - Пользователь нажимает "Создать комнату"
   - Сервер возвращает `room-created`
   - Вызывается `enterRoom(code)`
   - Модальное окно закрывается (если было открыто)
   - Показывается экран комнаты

2. **Подключение к комнате:**
   - Пользователь нажимает "Подключиться к комнате"
   - Открывается модальное окно
   - Пользователь вводит код и нажимает "Подключиться"
   - Сервер возвращает `room-joined`
   - Вызывается `enterRoom(code)`
   - Модальное окно закрывается
   - Показывается экран комнаты

### Сценарии с ошибками

1. **Комната не найдена:**
   - Пользователь вводит несуществующий код
   - Сервер возвращает `room-not-found`
   - `enterRoom()` не вызывается
   - Модальное окно остается открытым
   - Показывается ошибка в модальном окне

2. **Комната переполнена:**
   - Пользователь вводит код полной комнаты
   - Сервер возвращает `room-full`
   - `enterRoom()` не вызывается
   - Модальное окно остается открытым
   - Показывается ошибка в модальном окне

3. **Ошибка валидации кода:**
   - Пользователь вводит некорректный код (< 6 символов)
   - Валидация на клиенте предотвращает отправку
   - Модальное окно остается открытым
   - Показывается ошибка в модальном окне

## Преимущества подхода

1. **Централизация:** Изменение в одном месте (`enterRoom`) работает для всех сценариев входа в комнату
2. **Безопасность:** Закрытие происходит только при успешном входе в комнату
3. **Поддерживаемость:** Минимальное изменение кода, легко понять и поддерживать
4. **UX улучшение:** Автоматическое закрытие убирает лишнее действие

## Тестирование

### Unit-тесты (не требуются)
Изменение минимальное и не влияет на бизнес-логику. Функция `closeModal()` уже протестирована через интеграционные тесты.

### Manual тесты

1. **Создание комнаты:**
   - Нажать "Создать комнату"
   - Проверить, что модальное окно не открывается
   - Проверить, что экран комнаты показывается

2. **Подключение к комнате:**
   - Нажать "Подключиться к комнате"
   - Ввести корректный код
   - Нажать "Подключиться"
   - Проверить, что модальное окно закрывается
   - Проверить, что экран комнаты показывается

3. **Комната не найдена:**
   - Нажать "Подключиться к комнате"
   - Ввести несуществующий код
   - Нажать "Подключиться"
   - Проверить, что модальное окно остается открытым
   - Проверить, что ошибка показана

4. **Комната переполнена:**
   - Создать комнату
   - Подключиться к комнате первым пользователем
   - Подключиться к комнате вторым пользователем
   - Попытаться подключиться третьим пользователем
   - Проверить, что модальное окно остается открытым
   - Проверить, что ошибка показана

5. **Валидация кода:**
   - Нажать "Подключиться к комнате"
   - Ввести код длиной < 6 символов
   - Проверить, что модальное окно остается открытым
   - Проверить, что ошибка показана

## Риски

Нет значительных рисков. Изменение минимальное и обратимое.

## Совместимость

Изменение обратно совместимо с текущей функциональностью. Не влияет на существующие API или контракты.
