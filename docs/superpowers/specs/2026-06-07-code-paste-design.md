# Улучшение вставки кода комнаты

## Проблема

При подключении к комнате пользователь вводит 6-символьный код через 6 отдельных полей `<input maxlength="1">`. Вставка из буфера обмена не работает — весь код попадает только в первое поле.

## Решение

Заменить 6 отдельных полей на одно текстовое поле с `maxlength="6"`.

## Изменения

### src/code.js
- Добавить функцию `sanitizeCodeInput(value)` — приводит к uppercase, фильтрует `[^A-Z0-9]`, обрезает до 6 символов

### public/index.html
- Удалить `<div class="code-inputs">` с 6 полями
- Добавить одно `<input>` на всю ширину карточки

### public/style.css
- `.code-input`: ширина `100%` вместо `44px`, `letter-spacing: 0.3em` для визуального разделения символов
- Удалить/адаптировать `.code-inputs` (контейнер больше не нужен, или оставить как обёртку)

### public/app.js
- Удалить `CODE_INPUTS`, `setupCodeInputs()`, `getCodeFromInputs()`, `clearCodeInputs()`
- `openModal()`: очищать поле и ставить фокус
- `handleJoinRoom()`: читать код из одного поля
- Назначить `input` и `keydown` обработчики напрямую на поле в `init()`
- При вводе: `toUpperCase()`, фильтр `[^A-Z0-9]`, обрезка до 6 символов

## Тестирование

- `src/code.js`: тесты для `sanitizeCodeInput()` — uppercase, фильтрация невалидных символов, обрезка до 6, обработка null/undefined
- Серверная валидация кода (`isValidCode`) не меняется
- Ручная проверка: вставка 6-символьного кода, вставка >6 символов, невалидные символы, код с дефисами "ABC-234"
